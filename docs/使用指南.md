## 1. 项目概览

本项目基于 **Vibe-Trading** 框架开发 A 股量化选股策略，最终部署到 **国金 QMT 极速策略交易系统** 运行。

### 技术栈

| 组件 | 说明 |
|------|------|
| **Vibe-Trading** | AI 驱动的策略开发框架，输入自然语言，输出可执行策略 |
| **Python 3.11** | WSL 环境（Ubuntu 24.04），用于策略开发与回测 |
| **akshare** | A 股行情数据源（替代 Tushare Pro） |
| **东方财富数据中心** | 大宗交易、机构专用数据查询 |
| **QMT Python 3.6.8** | Windows 端生产环境（国金 QMT 内置） |
| **NumPy** | 所有策略计算使用 NumPy（QMT 环境 Pandas 不可用） |

### 网络环境

- 代理：`http://192.168.0.107:1082`
- 国内金融数据 API（东方财富、新浪）不走代理

## 4. 策略开发流程

### 4.1 标准流程

```
┌──────────────────────────────────────────────────────────────┐
│  1. 明确策略逻辑                                              │
│     - 选股条件（指标、技术形态、基本面）                          │
│     - 入场条件（何时买入）                                      │
│     - 出场条件（止盈、止损、时间、指标）                          │
│     - 仓位管理（固定权重 / 动态权重）                           │
│     - 市场过滤（大盘择时）                                      │
├──────────────────────────────────────────────────────────────┤
│  2. 生成策略提示词 → 写入 strategy_prompt.txt                  │
│     - 明确每个条件的中文描述                                     │
│     - 明确回测时间范围、基准                                     │
│     - 明确数据源（akshare / 东方财富）                          │
├──────────────────────────────────────────────────────────────┤
│  3. 运行 Vibe-Trading                                         │
│     $ source ~/vibe-env/bin/activate                          │
│     $ vibe-trading -f ~/VT/strategy_prompt.txt                │
├──────────────────────────────────────────────────────────────┤
│  4. 检查回测结果                                               │
│     - 查看收益率、最大回撤、夏普比率                               │
│     - 查看净值曲线、交易记录                                     │
│     - 如果结果不合理，迭代优化                                   │
├──────────────────────────────────────────────────────────────┤
│  5. 验证与调优                                                 │
│     - 用 verify_qmt.py 做分钟级模拟验证                        │
│     - 对比信号引擎 vs 日线收盘 vs 分钟低点代理                    │
├──────────────────────────────────────────────────────────────┤
│  6. 部署到 QMT                                                │
│     - 将策略拷贝到 Windows QMT 目录                             │
│     - 修改 stock_pool_qmt.txt                                  │
│     - 在 QMT 终端运行回测或实盘                                 │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 条件描述规范

写策略提示词时，对每个条件要**精确描述**：

| 模糊写法 | 精确写法 |
|---------|---------|
| "前一日下跌" | "昨日收盘价 < 昨日开盘价（收阴线）" |
| "5日均线上穿10日线" | "昨日收盘5日均线 > 昨日收盘10日均线，且前日5均线 <= 前日10均线" |
| "最近5日涨幅大于3%" | "昨日收盘价 / 6日前收盘价 - 1 > 3%" |
| "折溢价率大于4%" | 明确是折价（负值）还是溢价（正值） |
| "放量" | "昨日成交量 > 20日均量" |

### 4.3 常见调优方向

当回测结果不理想时，按顺序尝试：

1. **放宽入场条件** — AND 改 OR，降低门槛，增加信号数量
2. **增加过滤条件** — 成交量确认、相对强度等
3. **市场择时** — 添加大盘 > 60日均线判断
4. **动态仓位** — 按排名分配权重，让优者更重
5. **出场优化** — ATR 跟踪止损（让利润奔跑）+ 时间止损（防回撤）

### 4.4 迭代记录

```
v1 (原始条件)   →  -99.33%  (条件过严，几乎无信号)
v2 (放宽条件)   →   -9.04%  (信号增多，但胜率低)
v3 (加量过滤)   → +164.90%  (质量提升)
v4 (大盘择时)   → +922.65%  (牛市做多，熊市空仓)
```

## 6. 回测

### 6.1 回测引擎核心要点

- **入场价格**：用 `low[today] < close[yesterday]` 代理分钟级买入（比用 `close[today]` 更真实）
- **出场价格**：用 `close[today]`（QMT 用 `close[today]` 平仓）
  - ATR 跟踪止损用 `high[today]` 判断触发
- **费用**：买入万1.5，卖出万1.5+印花税万5
- **T+1 规则**：当日买入不可卖出
- **仓位管理**：按可用资金分配，非初始资金

### 6.2 回测脚本结构

```python
# 1. 加载数据（从 .cache_hs300/ 读取 CSV）
# 2. 转换为 NumPy 数组（[close, open, high, low, volume]）
# 3. 按日期逐日扫描
#    - 定期扫描条件（每周/每半月）
#    - 每天检查入场触发（低点代理）
#    - 每天检查出场条件
# 4. 计算每日净值
# 5. 输出报告（收益率、最大回撤、夏普率）
```

### 6.3 关键指标说明

| 指标 | 含义 | 参考值 |
|------|------|--------|
| 年化收益率 | 年化复合增长率 | > 30% 优秀 |
| 最大回撤 | 历史最大亏损幅度 | < 20% 良好 |
| 夏普比率 | 风险调整后收益 | > 1.5 优秀 |
| 胜率 | 盈利交易占比 | > 40% 良好 |
| 交易次数 | 总交易频率 | 年化 > 50 次 |

## 7. QMT 部署

### 7.1 环境限制（必须遵守）

| 限制 | 说明 |
|------|------|
| Python 版本 | 3.6.8（不支持 f-string、类型注解） |
| **禁止 import pandas** | QMT 内置 Pandas 崩溃（`__reduce_cython__` 错误） |
| 仅 3 个函数 | `init(C)`、`handlebar(C)`、`get_holdings(acct, type)` |
| **编码** | 文件头必须写 `#coding:gbk` |
| 股票代码格式 | `CODE.SH/SZ`（如 `000001.SZ`、`600519.SH`） |
| T+1 | 当日买入股 count 在持仓中但不可卖 |

## 8. 常见问题

### Q: Vibe-Trading 回测收益极高（+1000%+），但 QMT 只有 +50%
**原因**：入场价格假设不同。Vibe-Trading 默认用 `prev_close` 同根 K 线买入，实际 QMT 用 `close[today]` 或次日开盘。

**解决**：参考 [回测偏差说明](#todo) 中的低点代理法。

### Q: `gh` 命令报网络超时
```bash
export https_proxy=http://192.168.0.107:1082
export http_proxy=http://192.168.0.107:1082
```

### Q: Tushare 返回 "没有接口(daily)访问权限"
改用 akshare 作为数据源，见 [5.1 节](#51-个股日线数据akshare)。

### Q: QMT 策略运行时报 `AttributeError: 'Context' object has no attribute '...'`
检查函数名是否为 `init`、`handlebar`、`get_holdings` 三者之一，辅助函数必须内联在 `handlebar` 内。

### Q: 回测结果全是 -99%
入场条件太严格，几乎没有信号。先检查 `avg_positions`（平均持仓数），如果 < 1 说明条件过严。
